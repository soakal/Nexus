# Claude + AI Digest — 2026-07-22

**Claude Code stopped auto-invoking `/verify` and `/code-review`** *(New capability, high confidence — primary source, official CHANGELOG.md, v2.1.215)*. Previously Claude could decide on its own to run these two skills mid-session; now they only fire when explicitly slash-invoked. Tie-in: worth knowing for Codex Setup/Codex verify, since any workflow that depended on Claude self-triggering a verify pass now needs an explicit `/verify` or `/code-review` call — silent auto-verification is gone.

**New `sandbox.filesystem.disabled` setting** *(New capability, high confidence — primary source, official CHANGELOG.md, v2.1.216)*. Lets you opt a session out of filesystem sandboxing entirely rather than only relaxing permission prompts. Also in the same run of releases (v2.1.217): transcript-write failures now warn instead of silently losing data, and a memory leak in truncated MCP tool output was fixed — minor but worth knowing if a long NEXUS or Council Loop session ever felt like it degraded over time.

**`&` prefix teleports a CLI task straight to Claude Code Web, `--teleport` pulls it back** *(New technique, medium-high confidence — sourced from Anthropic Claude Code lead Boris Cherny's own post, not yet cross-checked against the official docs page)*. Prefixing a prompt with `&` in the local CLI hands the task to Claude's cloud infra and keeps running it even if the terminal closes or the machine sleeps; `--teleport <session-id>` later pulls that session — full history and context — back to any local machine. Distinct from the scheduled-routine setup already in use: this is ad hoc, one-off offloading of something that turns out to be longer than expected, without pre-configuring a schedule. Tie-in: handy for a Council Loop or NEXUS task that turns out to run long — bounce it to the cloud mid-session instead of babysitting a terminal.

No project tie-in — flagged purely as a notable containment/safety incident from a competing lab.
