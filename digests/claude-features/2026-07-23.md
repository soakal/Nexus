# Claude + AI Digest — 2026-07-23

**`/code-review` now runs as a background subagent, and subagents got a default concurrency cap** *(New capability, high confidence — primary source, official CHANGELOG.md, v2.1.218 and v2.1.217)*. `/code-review` no longer occupies your foreground conversation while it runs (v2.1.218). Separately, v2.1.217 added a default cap of 20 concurrent subagents (env-var configurable) and changed subagents to no longer spawn nested subagents by default. Tie-in: worth checking against Council Loop and any Workflow-tool fan-outs — if a run ever relied on deep subagent nesting or >20-way concurrency, the new defaults could silently throttle it.

**A 64-agent implementer/reviewer swarm rewrote Bun (~535K-1M lines) from Zig to Rust in 11 days for ~$165K** *(New technique, medium-high confidence — Simon Willison's July 19 post plus corroborating coverage from MarkTechPost, BigGo, and others; the underlying rewrite itself happened weeks earlier, Willison's post is what surfaced it now)*. Bun's founder ran a large parallel fleet of Claude instances split into implementer and reviewer roles rather than one agent iterating alone, and shipped a 1-2% throughput gain plus a ~10x memory-usage drop under stress. Tie-in: this is Council Loop's Arbiter/Engineer/Realist split, just run at a scale of dozens of agents instead of three — worth thinking about whether any Council Loop or Workflow job would benefit from a wider implementer/reviewer fleet instead of a single engineer pass.

**That same Bun rewrite drew real backlash — Zig's creator called it "unreviewed slop"** *(Claude buzz, medium confidence — reported by devclass.com covering the creator's public comments; this is dev-community controversy, not viral social reach)*. The criticism centers on trusting a large AI-driven migration of security-relevant systems code without the review depth a human-led rewrite would normally get. No project tie-in — flagged as a reminder that fast agentic migrations invite exactly this kind of scrutiny, relevant context if Council Loop or SOPForge output is ever judged by outside reviewers.

**Google DeepMind launched Gemini 3.5 Flash Cyber, a dedicated vulnerability-hunting model** *(Other AI, high confidence — primary source, DeepMind's own blog, corroborated by TheHackerNews and Help Net Security, July 21-22)*. Built on Gemini 3.5 Flash, Google says it "significantly" outperformed Gemini 3.5/3.6 Flash and Claude Opus 4.6 at finding new vulnerabilities in stress tests against Chrome and Safari (55 confirmed V8 issues), but it's being kept to a limited pilot for governments and trusted partners via CodeMender rather than general release. No project tie-in — flagged as a significant competitive move in the security-focused-model space, notable since NEXUS's own action broker and Hermes allowlist are exactly the kind of security-conscious design this class of model would eventually be used to probe.
