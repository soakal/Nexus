# Claude Features Digest — 2026-07-12

**/doctor is now an active auto-repair checkup, not just a report** *(New capability)*. It now checks version/auth/connectivity, dedupes personal vs. project CLAUDE.md files, offers to split an overgrown root CLAUDE.md into nested files/skills, flags slow hooks, and pre-approves frequently-denied read-only commands — pressing `f` applies fixes with confirmation. Worth a run against this NEXUS CLAUDE.md (it's large) and the Council Loop/SOPForge repos to see what it flags. First step: run `/doctor` here and see what it proposes for the CLAUDE.md split.

**Auto Mode gained safety guardrails** *(New capability)*: it now blocks any attempt to tamper with session transcript files and asks for confirmation before running `rm -rf` on a path/variable it can't resolve from context. Relevant if Council Loop's auto-commit-on-accept loop ever runs under Auto Mode rather than the custom `/loop` driver — worth checking whether that loop would hit the new rm -rf confirmation prompt and stall unattended.

**MCP server context cost is asymmetric with skills** *(New technique)*: a skill costs ~30-50 tokens until invoked, but a handful of connected MCP servers can cost 50k+ tokens of context up front, plus a small per-request overhead, whether or not you call them that session. Brian runs several always-on MCP servers (Obsidian vault, Chrome automation, Gmail/Calendar) — running `/mcp` per-project to disable ones not needed for that session's task could meaningfully cut baseline context/cost on NEXUS or SOPForge sessions that don't touch email/calendar. First step: `/mcp` in a NEXUS-only session and disable Gmail/Calendar/Chrome if unused that session.

**SubagentStop hooks as a quality gate, separate from a reviewer role** *(New technique)*: rather than only relying on a review/critic agent (as Council Loop's Realist role does), a `SubagentStop` hook can programmatically gate a subagent's output (e.g., require a schema, a passing test command, or a specific string) before the result is accepted back into the parent loop — a cheaper, deterministic check that composes with an LLM reviewer rather than replacing it. Could tighten the Agent/Workflow tool usage in this repo where a subagent's returned text currently isn't structurally validated before being trusted. First step: try a `SubagentStop` hook on one narrow Agent-tool call (e.g. enforcing a JSON shape) to see how it interacts with the existing review layer.

Nothing else in the lookback cleared the bar — the rest of this week's Claude Code changes (JetBrains flicker fix, launcher-symlink fix, Bedrock/Vertex/Foundry Auto Mode default, Trusted Devices for Remote Control) are either bug fixes or scoped to Team/Enterprise/non-direct-API usage that doesn't apply to how NEXUS/Council Loop/SOPForge bill through the direct Anthropic API.
